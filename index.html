Creating a PD code

  1. Pick a starting point & direction to travel
  2. Label each edge
  3. Write a tuple for each crossing
  • PD code = numerical representation of a diagram
  • direction to travel = orientation
  • stop at UNDERPASS to write tuple

Reidemeister moves

Theorem

Two knot diagrams belonging to the same knot, up to planar isotopy, can be related by a sequence of the three Reidemeister moves.
This is important b/c it allows us to manipulate a diagram without changing the knot itself, which is key to our project.

Simplify with Reidemeister move type 1

  • 4 diagram segment arragments which can be simplified by RM1
  • the corresponding PD code tuple has at most 3 unique values (b/c the loop value is repeated)

The usual label changes

x → x, x<b
x → x-2, x>b
  • b is label of edge which is loop
  • Labels of edges before b do not change
  • Labels of edges after b decrease by 2 (b/c of edges which are merged together)

...unless b=1

Applying the usual label changes:

x → x, x<1
x → x-2, x>1

Instead:

x → x-2, x>2
x → m-2, x=2
problem
  • Edge has 2 labels b/c 2 & m map to different values
solution
  • map 2 to same value as m (m-2)

...or b=m

Applying the usual label changes:

x → x, x<m
x → x-2, x>m

Instead:

x → x, x≤m-2
x → 1, x=m-1
problem
  • Edge has 2 labels b/c 1 & m-1 map to different values
solution
  • map m-1 to same value as 1 (1)

Simplify with Reidemeister move type 2

2 patterns to PD code tuples
  • 2nd elements equal & 3rd and 1st elements equal
  • 3rd and 1st elements equal & 4th elements equal

The usual label changes

x → x, x<n1
x → x-2, n1<x<n2
x → x-4, x>n2
  • n1 - least label of edge involved in the loop
  • n2 - greatest label of edge involved in the loop
  • labels of edges before n1 are not affected
  • labels of edges after n1 decrease by 2 b/c of merged edges
  • labels of edges after n2 decrease by additional 2 (for a totla of 4) b/c of more merged edges

...unless edge 1 is involved

Applying the usual label changes:

x → x, x<1
x → x-2, 1<x<n2
x → x-4, x>n2

Instead:

x → m-4, x=2
x → x-2, 2<x<n2
x → x-4, x>n2
problem
  • labels 2 and m are mapped to different values
solution
  • map 2 to the same value as m (m-4)
